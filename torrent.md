# BitTorrent: Архитектура и принципы работы протокола

BitTorrent — это протокол для обмена файлами через интернет, построенный по принципу **P2P (peer-to-peer)**. В отличие от традиционной модели «клиент-сервер» (HTTP/FTP), где нагрузка ложится на один мощный сервер, BitTorrent распределяет задачу по распространению контента между всеми участниками сети.

## 1. Фундаментальные концепции

### Терминология
*   **Peer (Пир):** Участник, который обменивается данными.
*   **Seeder (Сид):** Пир, обладающий полной копией файла и распространяющий его.
*   **Leecher (Личер):** Пир, который находится в процессе загрузки файла. Если личер начинает раздавать уже скачанные части, его называют *Peer*.
*   **Swarm (Рой):** Группа всех пиров, участвующих в обмене одного конкретного торрента.
*   **Tracker (Трекер):** Сервер, который координирует работу пиров в рое. Он не хранит файлы, а лишь помогает пирами находить друг друга.

### Принцип фрагментации
Файл разбивается на небольшие части, называемые **pieces**. Размер части (от 16 КБ до нескольких МБ) определяется при создании торрента. Для каждой части рассчитывается **контрольная сумма (SHA-1 хэш)**. Это гарантирует целостность данных: если пакет будет поврежден, клиент легко обнаружит ошибку и перезапросит фрагмент снова.

## 2. Процесс взаимодействия

Типичный жизненный цикл торрента состоит из четырех этапов:

### 1. Метаинформация (.torrent файлы)
Торрент-файл — это закодированная структура (Bencode), не содержащая данных файла, но содержащая:
*   Адрес трекера.
*   Имена файлов и размеры.
*   Хэши всех частей (pieces) для проверки целостности.

### 2. Регистрация (Handshake с трекером)
Клиент отправляет хэш-идентификатор торрента (**info_hash**) на трекер. В ответ трекер присылает список IP-адресов других пиров (сидов и личеров), участвующих в скачивании этого файла.

### 3. P2P Обмен данными (Choking/Unchoking)
Когда клиент узнает адреса других пиров, он связывается с ними напрямую (TCP или uTP).
*   **Bitfield:** Пиры обмениваются сообщениями о том, какие части файла у кого есть.
*   **Tit-for-Tat (Принцип «ты — мне, я — тебе»):** Механизм борьбы с «паразитами» (теми, кто только качает, но не раздает). Клиент отдает данные тем, кто отдает ему с наибольшей скоростью.

### 4. Сборка
После получения всех фрагментов клиент проверяет их через контрольные суммы, собирает их в цельный файл и меняет статус с *Leecher* на *Seeder*.

## 3. Децентрализация: DHT и PEX

Современные клиенты могут обходиться без центрального трекера благодаря двум технологиям:

*   **DHT (Distributed Hash Table):** Каждый клиент становится «мини-трекером». Пиры хранят информацию о том, у кого находится часть данных, в распределенной хеш-таблице (протокол Kademlia).
*   **PEX (Peer Exchange):** Метод, при котором пиры обмениваются списками известных им адресов друг с другом напрямую, распространяя информацию о «живых» участниках роя.
*   **LPD (Local Peer Discovery):** Поиск пиров в локальной сети, позволяющий передавать данные на сверхвысоких скоростях минуя интернет.

## 5. Резюме
BitTorrent — это элегантное решение задачи масштабируемости. Чем больше людей скачивает файл, тем **выше** становится общая скорость сети, так как каждый скачавший автоматически становится узлом раздачи. Именно это делает протокол незаменимым инструментом для дистрибуции тяжелых данных в современной IT-инфраструктуре.
