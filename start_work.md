## 1. Установка Python 3.12

Самый простой и рекомендуемый способ установки Python на macOS — использование **Homebrew**, менеджера пакетов для macOS.

### 1.1. Установка Homebrew (если еще не установлен)

1.  Откройте приложение **Терминал** (Terminal.app). Его можно найти в `Программы` -> `Утилиты` или через Spotlight (Cmd + Пробел, затем введите "Терминал").
2.  Скопируйте и вставьте следующую команду в Терминал и нажмите Enter:

    ```bash
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
    ```
3.  Следуйте инструкциям на экране. Вам может потребоваться ввести пароль администратора.
4.  После завершения установки Homebrew, выполните команды, которые предложит установщик для добавления Homebrew в ваш PATH. Обычно это что-то вроде:

    ```bash
    echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
    eval "$(/opt/homebrew/bin/brew shellenv)"
    ```
    *(Примечание: Путь `/opt/homebrew/bin/` может отличаться для старых версий macOS или процессоров Intel. Если команда `brew doctor` выдаст ошибку, следуйте ее рекомендациям.)*

### 1.2. Установка Python 3.12

1.  Откройте Терминал.
2.  Убедитесь, что Homebrew обновлен:

    ```bash
    brew update
    ```
3.  Установите Python 3.12:

    ```bash
    brew install python@3.12
    ```
4.  Homebrew установит Python и связанные с ним инструменты (pip, venv). После установки проверьте версию Python:

    ```bash
    python3 --version
    ```
    Вы должны увидеть `Python 3.12.x`.

## 2. Установка PyCharm

PyCharm — это мощная интегрированная среда разработки (IDE) для Python.

### 2.1. Скачивание PyCharm

1.  Перейдите на официальный сайт JetBrains: [https://www.jetbrains.com/pycharm/download/](https://www.jetbrains.com/pycharm/download/)
2.  Выберите версию для macOS. Вам будет предложено скачать **Community Edition** (бесплатная, подходит для большинства задач) или **Professional Edition** (платная, с дополнительными функциями для веб-разработки и др.). Нажмите на кнопку загрузки соответствующей версии.

### 2.2. Установка PyCharm

1.  После загрузки файла `.dmg` откройте его.
2.  В появившемся окне вы увидите иконку PyCharm и ярлык папки `Applications`.
3.  Перетащите иконку PyCharm в папку `Applications`.
4.  Теперь вы можете запустить PyCharm из папки `Программы` (Applications) или через Spotlight.
5.  При первом запуске PyCharm может предложить импортировать настройки, создать рабочий стол или выполнить другие начальные настройки. Следуйте инструкциям на экране.

## Git

macOS comes with a pre-installed version of [Git](http://git-scm.com/), but we'll install our own through Homebrew to allow easy upgrades and not interfere with the system version. To do so, simply run:

```
brew install git
```

После завершения установки проверьте, что все прошло успешно:

```
which git
```
Результат должен быть таким: `/usr/local/bin/git`.

Теперь выполним базовую настройку. Скачайте файл [.gitconfig](https://raw.githubusercontent.com/nicolashery/mac-dev-setup/master/.gitconfig) в свою домашнюю директорию:

```
cd ~
curl -O https://raw.githubusercontent.com/nicolashery/mac-dev-setup/master/.gitconfig
```
Этот файл добавит цвета в вывод команд Git: status, branch и diff, а также настроит несколько псевдонимов (алиасов). Вы можете просмотреть содержимое файла и дополнить его по своему усмотрению.

Далее необходимо указать ваши данные пользователя Git (они должны совпадать с именем и адресом электронной почты, которые вы используете на GitHub и Heroku [GitHub](https://github.com/) and [Heroku](http://www.heroku.com/):

```
git config --global user.name "Your Name Here"
git config --global user.email "your_email@youremail.com"
```

Эти данные будут добавлены в ваш файл .gitconfig.

На Mac важно не забывать добавлять .DS_Store (скрытый системный файл macOS, который отображается в папках) в файлы .gitignore ваших проектов. Также рекомендуется настроить глобальный файл .gitignore, расположив его, например, в домашней директории (но вам нужно убедиться, что ваши соавторы по проекту сделают то же самое):
```
cd ~
curl -O https://raw.githubusercontent.com/nicolashery/mac-dev-setup/master/.gitignore
git config --global core.excludesfile ~/.gitignore
```

### Нужно сгенирировать ssh-ключ для подключения к репо.

Статья: https://selectel.ru/blog/tutorials/how-to-generate-ssh/

#### 1. Генерация SSH-ключа

Открой терминал и выполни:

```bash
ssh-keygen -t ed25519 -C "your_email@example.com"
```
Если система не поддерживает ed25519, используй RSA:

```bash
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
```
Что делать дальше:

- На запрос Enter file... — нажми Enter (стандартный путь ~/.ssh/id_ed25519).
- На запрос Enter passphrase — введи пароль (или оставь пустым, но с паролем — безопаснее).

✅ Готово. У тебя появились 2 файла:

- id_ed25519 — приватный (НИКОГДА не передавай)
- id_ed25519.pub — публичный (можно и нужно показывать)

2. Добавление ключа в SSH-агент
3. 
- Запусти агент и добавь ключ:

```bash
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519
```

3. Подключение к GitHub

3.1. Скопируй публичный ключ
```bash
cat ~/.ssh/id_ed25519.pub
Скопируй весь вывод (начинается с ssh-ed25519 ...).
```

3.2. Добавь ключ в GitHub

Зайди в Settings → SSH and GPG keys.

- Нажми New SSH Key.
- Вставь ключ, дай понятное название (например, Work Laptop).
- Нажми Add SSH Key.

3.3. Проверь подключение

```bash
ssh -T git@github.com
```
Если увидишь:

```text
Hi <username>! You've successfully authenticated...
```


## 3. Установка Docker

Docker — это платформа для разработки, доставки и запуска приложений в контейнерах.

### 3.1. Скачивание Docker Desktop

1.  Перейдите на официальный сайт Docker: [https://www.docker.com/products/docker-desktop/](https://www.docker.com/products/docker-desktop/)
2.  Нажмите кнопку "Download for Mac".
3.  Выберите версию для `Apple Chip` (для Mac с процессорами M1/M2/M3) или `Intel Chip` в зависимости от вашего Mac.

### 3.2. Установка Docker Desktop

1.  Откройте загруженный файл `.dmg`.
2.  Перетащите иконку Docker в папку `Applications`.
3.  Запустите Docker из папки `Программы`.
4.  При первом запуске Docker Desktop может потребовать предоставить разрешения и установить дополнительные компоненты. Следуйте инструкциям на экране. Вам может потребоваться ввести пароль администратора.
5.  После установки Docker Desktop, в строке меню (верхняя часть экрана) появится иконка Docker. Нажмите на нее, чтобы убедиться, что Docker запущен.

### 3.3. Проверка установки Docker

1.  Откройте Терминал.
2.  Выполните команду, чтобы проверить версию Docker:

    ```bash
    docker --version
    ```
3.  Выполните команду, чтобы проверить версию Docker Compose:

    ```bash
    docker-compose --version
    ```

## 5. Работа с библиотеками Python

После установки Python вы можете управлять библиотеками (пакетами) с помощью менеджера пакетов `pip`.

### 5.1. Установка библиотек

1.  Откройте Терминал.
2.  Для установки библиотеки используйте команду `pip install`. Например, чтобы установить библиотеку `requests` для работы с HTTP-запросами:

    ```bash
    pip install requests
    ```
    *Примечание: Если у вас несколько версий Python, может потребоваться использовать `pip3` вместо `pip`, или `python3 -m pip install ...`.*

3.  Чтобы установить конкретную версию библиотеки:

    ```bash
    pip install requests==2.28.1
    ```

### 5.2. Обновление библиотек

Для обновления библиотеки до последней версии:

```bash
pip install --upgrade requests
```

### 5.3. Удаление библиотек
Чтобы удалить библиотеку:

```bash
pip uninstall requests
```

### 5.4. Просмотр установленных библиотек
Чтобы увидеть список всех установленных библиотек в текущей среде:
```bash
pip list
```
### 5.5. Виртуальные окружения (venv)
Рекомендуется использовать виртуальные окружения для изоляции зависимостей проектов.

Создание виртуального окружения:
Перейдите в папку вашего проекта в Терминале и выполните команду:

```bash
python3 -m venv .venv
(где .venv — имя папки виртуального окружения, часто используется именно это имя)
```
Активация виртуального окружения:

```bash
source .venv/bin/activate
(После активации в начале строки Терминала появится (.venv))
```

Деактивация виртуального окружения:
```bash
deactivate
(После деактивации (.venv) исчезнет)
```

### 6. Базовые команды macOS (Терминал)
Вот несколько основных команд, которые пригодятся при работе в Терминале macOS.

### 6.1. Навигация по файловой системе

```bash
pwd — Показать текущую директорию.
Пример: pwd
ls — Показать содержимое текущей директории.
ls -l — Показать подробный список (права доступа, владелец, размер, дата).
ls -a — Показать все файлы, включая скрытые (начинающиеся с точки).
cd <путь> — Сменить директорию.
cd Documents — Перейти в папку Documents.
cd .. — Перейти на уровень выше.
cd ~ — Перейти в домашнюю директорию.
cd / — Перейти в корневую директорию файловой системы.
```

### 6.2. Работа с файлами и директориями

```bash
mkdir <имя_папки> — Создать новую директорию.
Пример: mkdir my_project
touch <имя_файла> — Создать новый пустой файл.
Пример: touch index.html
cp <источник> <назначение> — Скопировать файл или директорию.
cp file.txt new_file.txt — Скопировать файл.
cp -r folder/ another_folder/ — Скопировать директорию рекурсивно.
mv <источник> <назначение> — Переместить или переименовать файл/директорию.
mv old_name.txt new_name.txt — Переименовать.
mv file.txt ../destination_folder/ — Переместить.
rm <имя_файла> — Удалить файл.
rm -r <имя_папки> — Рекурсивное удаление каталога (используйте с осторожностью!).
rm -rf <имя_папки> — Рекурсивное удаление каталога без запроса подтверждения (крайне осторожно!).
```

### 6.3. Просмотр содержимого файлов
```bash
cat <имя_файла> — Вывести содержимое файла на экран.
less <имя_файла> — Постраничный просмотр файла (нажмите q для выхода).
head <имя_файла> — Показать первые 10 строк файла.
tail <имя_файла> — Показать последние 10 строк файла.
```

### 6.4. Поиск

grep <шаблон> <имя_файла> — Поиск строк, содержащих шаблон, в файле.
Пример: grep "error" log.txt

### 6.5. Управление процессами
```bash
top — Показать список запущенных процессов и использование ими ресурсов (нажмите q для выхода).
kill <PID> — Завершить процесс по его идентификатору (PID).
```

### 6.6. Получение справки
```bash
man <команда> — Показать руководство (man page) по команде.
Пример: man ls (нажмите q для выхода).
```
