# 🐳 Docker: Сборка, управление и отладка

---

## 1. Основные команды для работы с контейнерами

| Команда | Описание |
| :--- | :--- |
| `docker run <image>` | Запустить контейнер из образа |
| `docker run -d <image>` | Запустить в фоне (detached mode) |
| `docker run -it <image> bash` | Интерактивный режим (bash) |
| `docker run -p 8080:80 <image>` | Проброс портов (хост:контейнер) |
| `docker run -v host:cont <image>` | Подключить том (volume) |
| `docker run --name <name> <image>` | Запуск с конкретным именем |
| `docker start/stop/restart <id>` | Управление состоянием контейнера |
| `docker rm -f <id>` | Принудительное удаление |
| `docker ps -a` | Список всех контейнеров (в т.ч. остановленных) |
| `docker logs -f <id>` | Логи в реальном времени |
| `docker exec -it <id> bash` | Зайти в запущенный контейнер |
| `docker inspect <id>` | Подробная информация (JSON) |
| `docker stats` | Статистика по ресурсам (CPU/RAM) |

## 2. Работа с образами, томами и сетями

### Образы (Images)
* `docker images` — список локальных образов.
* `docker pull <image>` — скачать образ из репозитория.
* `docker build -t <name> .` — собрать образ из Dockerfile.
* `docker rmi <image>` — удалить образ.
* `docker push <user>/<image>` — загрузить в Docker Hub.
* `docker save <image> > file.tar` — экспортировать образ.
* `docker load < file.tar` — импортировать образ.

### Тома (Volumes)
* `docker volume ls` — список томов.
* `docker volume create <name>` — создать том.
* `docker volume prune` — удалить все неиспользуемые тома.

### Сети (Networking)
* `docker network ls` — список сетей.
* `docker network create <name>` — создать сеть.
* `docker network connect <net> <cont>` — подключить контейнер к сети.

## 3. Docker Compose

| Команда | Описание |
| :--- | :--- |
| `docker-compose up` | Запустить сервисы |
| `docker-compose up -d` | Запустить в фоне |
| `docker-compose down` | Остановка и удаление контейнеров |
| `docker-compose build --no-cache` | Пересборка без использования кэша |
| `docker-compose ps` | Список сервисов |
| `docker-compose logs -f <service>` | Логи конкретного сервиса |
| `docker-compose exec <service> bash` | Зайти в контейнер сервиса |

**Пример `docker-compose.yml` (Python + Postgres):**
```yaml
version: '3.8'
services:
  web:
    build: .
    ports: ["8000:8000"]
    volumes: [".:/app"]
    depends_on: [db]
  db:
    image: postgres:13
    environment:
      POSTGRES_PASSWORD: example
```

Запуск: docker-compose up -d

### Алгоритм запуска и отладки

**Чистая сборка и запуск**

1. Для исключения проблем с битыми слоями кэша используйте:
```bash
docker compose build --no-cache
docker compose up -d
```
или

```bash
# Первый запуск
git clone <repo>
cd сюда название проекта
cp .env.example .env   # заполнить переменные
docker-compose up --build -d

# Обновление
git pull
docker-compose up --build -d
```
   
2. Запуск с логами в реальном времени
С логами - хвостик смотрим: если хотим все то без --tail=, но будет огромное полотно
```bash
docker compose logs -f --tail=100 <имя_сервиса>
```

3. Автоматическая генерация сертификатов (nginx-le)
При использовании FROM `umputun/nginx-le:v1.4.3:` - чтобы не покупать а свои генерить.

обязательно укажите переменные: LEMAIL (email) и URLS (доменное имя).
```bash
пример:
    environment:
      - TZ=GMT
      - LE_EMAIL=olyabjj@gmail.com
      - LE_FQDN=hello.i.pro
      - LETSENCRYPT=true
```
Важно: порты 80 и 443 хоста должны быть свободны и перенаправлены в этот контейнер, иначе проверка Let's Encrypt завершится ошибкой.
4. Работа с Volumes (тома)
При добавлении новых volumes:

- Соблюдайте синтаксис: путь_на_хосте:путь_в_контейнере.
- Ошибка доступа: Если сервис не запускается, проверьте права на папку хоста (chown/chmod). Пользователь внутри контейнера должен иметь права на чтение/запись.
- Кэширование: Если изменения файлов на хосте не видны в контейнере, убедитесь, что вы не примонтировали том поверх важных системных папок проекта (например, /app/venv или /app/node_modules).
5. Настройка Networks
При использовании явного драйвера:
```bash
networks:
  default:
    driver: bridge
```
Связанность: Убедитесь, что все необходимые сервисы находятся в одной сети. По умолчанию docker compose помещает все в одну сеть, но при явном объявлении следите за тем, чтобы все сервисы web, db и др. имели ссылку на этот networks.
Диагностика: Если контейнеры не видят друг друга, выполните docker network inspect <имя_сети_проекта> и проверьте, какие контейнеры получили IP-адреса.

**Частые проблемы:**

- Порты заняты: sudo ss -tulpn | grep :<порт> — проверьте, не занят ли порт хоста другим процессом.
- Отказано в доступе: Проверьте права доступа ls -ld <папка_хоста>.
- Нет места на диске: df -h — если диск заполнен, Docker не сможет создать слои образа.
- Полный сброс: docker compose down -v — удалит контейнеры и анонимные тома (полезно при критических ошибках конфигурации).

Алгорит что-то надо посмотреть внутри контейнера - как зайти:
- опасть внутрь работающего контейнера и выполнить там команды

**Основная команда:**

```bash
docker exec -it <имя_или_id_контейнера> /bin/bash

Если в образе нет bash (часто бывает в Alpine Linux), используйте стандартную sh:

docker exec -it <имя_или_id_контейнера> /bin/sh
```

**Разбор флагов:**
-i (interactive) — оставляет стандартный ввод (STDIN) открытым, даже если вы не подключены к контейнеру.
-t (tty) — выделяет псевдотерминал. Без этого флага команды будут работать как в обычном терминале (без красивых шрифтов, истории команд и т. д.).

**Например нужно поковырять что в БД**

```bash
- docker-compose exec db psql -U postgres 
```
