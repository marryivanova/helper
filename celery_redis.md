# 📋 Шпаргалка: Выбор системы для распределенных задач

## 🎯 Когда нужна распределенная очередь задач?

**Пример:** `\src\celery_app`

### Типичные кейсы

| Кейс | Пример |
|------|--------|
| 🖼️ Обработка медиа | Ресайз изображений, конвертация видео, обработка документов |
| 📧 Отложенные уведомления | Отправка email, push-уведомлений, SMS |
| 📨 Массовые рассылки | Отправка сообщений сотням тысяч пользователей |
| 🔄 Периодические задачи | Генерация отчетов, очистка кеша, бэкапы |
| ⚡ Тяжелые вычисления | Обработка данных, машинное обучение |

---

## 🧠 Этап 1: Celery или Kafka?

### Сравнение

| Характеристика | Celery | Kafka |
|----------------|--------|-------|
| **Назначение** | Распределенная очередь задач | Платформа потоковой передачи данных |
| **Модель работы** | По запросу (request-response) | Постоянный поток событий |
| **Хранение истории** | Нет (задача выполняется и удаляется) | Да (можно перемотать историю) |
| **Сценарии** | Email, обработка изображений, отчеты | Логи, трекинг, метрики, аналитика |
| **Сложность** | Проще | Сложнее |

### 💡 Вывод

> **Celery** — для фоновых задач по запросу  
> **Kafka** — для обработки событийных потоков в реальном времени

**В реальных проектах их часто используют вместе!**

## 🎯 Этап 2: Какой брокер выбрать для Celery?

### 📊 Сравнение Redis vs RabbitMQ

| Характеристика | 🔴 Redis | 🐇 RabbitMQ |
|----------------|----------|-------------|
| **Надежность** | ⚠️ Слабая. Потеря данных при сбое | ✅ Высокая. Гарантия доставки (acks) |
| **Скорость** | ⚡ Очень быстрый (в памяти) | 🐢 Медленнее (запись на диск) |
| **Кластеризация** | 🔧 Сложная (Sentinel) | ✅ Встроенная HA кластеризация |
| **Установка** | ✅ Простая (одна команда) | 🔧 Сложнее (Erlang/AMQP) |
| **Маршрутизация** | 📌 Простая (очередь-ключ) | 🎯 Гибкая (exchanges/bindings) |
| **Поддержка** | 🆓 Бесплатный | 🆓 Бесплатный |

---

## ✅ Когда выбирать Redis

### Подходит для:

- ✅ Скорость важнее надежности
- ✅ Небольшие/средние проекты
- ✅ Разработка и тестирование
- ✅ Redis уже используется как кеш
- ✅ Задачи терпимы к потере

### Примеры задач:

```python
# Некритичные задачи
- Отправка email-рассылок
- Ресайз изображений
- Очистка кеша
- Сбор аналитики
- Отправка уведомлений
```

### ⚠️ Важно:

> Вы готовы к тому, что **небольшой процент задач может быть потерян** при сбое Redis

---

## ✅ Когда выбирать RabbitMQ

### Подходит для:

- ✅ Надежность критична
- ✅ Нужна сложная маршрутизация
- ✅ Высоконагруженный продакшн
- ✅ Требуется отказоустойчивость
- ✅ Нельзя потерять ни одной задачи

### Примеры задач:

```python
# Критичные задачи
- Обработка платежей
- Оформление заказов
- Финансовые транзакции
- Сложные бизнес-процессы
- Регистрация пользователей
```

### ⚠️ Важно:

> Если сервер перезагрузится — **каждая задача должна быть выполнена**

---

## 💡 Гибридный подход

### Лучшая комбинация

```python
# Broker: RabbitMQ (надежность)
# Result Backend: Redis (скорость)

app = Celery(
    'tasks',
    broker='pyamqp://guest@localhost//',      # RabbitMQ
    backend='redis://localhost:6379/0'        # Redis для результатов
)
```

**Преимущества:**
- ✅ Надежность RabbitMQ
- ✅ Скорость Redis для хранения результатов

## 🚀 Быстрый старт

### С Redis

```bash
# Установка Redis
sudo apt install redis-server

# Celery с Redis
pip install celery[redis]

# Настройка
app = Celery('tasks', broker='redis://localhost:6379/0')
```

### С RabbitMQ

```bash
# Установка RabbitMQ
sudo apt install rabbitmq-server

# Celery с RabbitMQ
pip install celery[rabbitmq]

# Настройка
app = Celery('tasks', broker='pyamqp://guest@localhost//')
```
